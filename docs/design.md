# 设计文档

## 总体思路

系统采用分层架构：仓库采集层负责获取源码与元数据，静态分析层负责抽取项目画像，索引检索层负责历史作品召回，智能体层负责分析与比对，报告层负责生成面向人类阅读的描述文档和比较文档。

## 数据流

1. 输入历史仓库或新提交仓库地址。
2. 采集 README、文档、目录树、源码文件和提交历史。
3. 静态分析源码，形成模块、函数、调用关系和关键机制描述。
4. 建立关键词索引和向量索引。
5. 智能体根据任务目标召回证据并生成报告。
6. 报告输出时保留源码路径、函数名和文档片段作为证据链。

## 关键约束

生成结论必须有证据支撑。无法从源码或文档确认的信息应标记为“未确认”，不得直接编造。
