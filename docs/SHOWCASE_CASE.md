# KernelSage 精选样例报告

本文档固定两条演示样例，用于答辩讲稿和演示视频。它不是自动生成报告的替代品，而是对自动报告进行人工审阅后的展示材料。更严格的人工校准样例见 `docs/GOLDEN_CASES.md` 和 `docs/golden/`，用于说明报告质量如何被审核。

两类样例分别覆盖：

| 样例 | 输入仓库 | 展示目标 |
| --- | --- | --- |
| 稳定性样例 | `xv6-public` | 展示系统对经典小型 OS 仓库的稳定识别、证据链和相似性比较 |
| 比赛场景样例 | `oskernel2024-aabcb` | 展示系统对公开比赛作品样本的分析、比较和边界处理 |

生成报告保存在 `data/reports/` 下，其中最终 Markdown/HTML 演示报告已随仓库提交，便于初赛评委直接查看；prompt、tmp 和缓存类运行产物仍不提交。本文档保留人工整理后的关键观察。

## 复现命令

```powershell
$env:PYTHONPATH='src'

python scripts\kernelsage.py describe data\samples\xv6-public --repo-id xv6-public
python scripts\kernelsage.py compare data\samples\xv6-public --repo-id xv6-public --limit 3

python scripts\kernelsage.py describe data\samples\oskernel2024-aabcb --repo-id oskernel2024-aabcb
python scripts\kernelsage.py compare data\samples\oskernel2024-aabcb --repo-id oskernel2024-aabcb --limit 3
```

对应本地报告：

| 类型 | 路径 |
| --- | --- |
| xv6 描述报告 | `data/reports/describe/xv6-public.md` |
| xv6 对比报告 | `data/reports/compare/xv6-public_vs_history.md` |
| 比赛样本描述报告 | `data/reports/describe/oskernel2024-aabcb.md` |
| 比赛样本对比报告 | `data/reports/compare/oskernel2024-aabcb_vs_history.md` |

### 人工点评

这个样例适合演示系统稳定性。xv6 的目录和机制较清晰，系统能稳定识别七个 OS 维度，并能把判断落到具体源码证据。对比样本选择也合理：既有同风格的 `os-tutorial`，也有同源教学路线的 `xv6-riscv`，还有不同路线的 `Little Kernel`。

需要强调的边界：

- `xv6-public` 是架构参考样本，不是比赛获奖样本。
- 代码级相似线索只说明“值得人工复核”，不是抄袭裁定。
- 创新性判断只在当前参考库范围内成立，不能泛化为全量 OS 生态结论。

- 代码级相似线索包含宏名、路径、结构体/类型和片段相似度等代表项。

### 人工点评

这个样例适合演示系统贴近比赛场景的能力。它能展示 KernelSage 如何处理真实比赛作品样本：先生成结构化画像，再从历史库中选择相近项目，最后输出功能重合、差异点和代码级复核线索。

需要强调的边界：

- `oskernel2024-aabcb` 是公开比赛作品样本，但当前没有核验获奖等级。
- 与其他比赛样本相似，不等于代码重复；需要结合完整代码和提交历史人工判断。
- 参考库当前有 21 个代表性样本，其中包含 3 个 2024 功能赛道一等奖样本，但创新性判断仍必须保守解释。

## 演示视频建议流程

1. 打开本文档，说明选择两个样例的原因：一个稳定，一个贴近比赛。
2. 展示复现命令，强调默认不调用 LLM、不消耗 API。
3. 打开 `xv6-public` 描述报告，讲七维 OS 机制和 self-check。
4. 打开 `xv6-public` 对比报告，讲历史样本选择、代码级线索和边界声明。
5. 打开 `oskernel2024-aabcb` 描述报告，讲比赛作品样本的识别结果。
6. 打开 `oskernel2024-aabcb` 对比报告，讲教学基线和比赛样本如何共同参与比较。
7. 收尾强调：KernelSage 是评审辅助工具，不自动裁定抄袭，不强行确认创新点，不把未核验样本硬标为获奖案例。

## 展示结论

这两条样例共同说明：

- 系统能完成“输入仓库 -> 描述报告 -> 对比报告 -> self-check -> 人工点评”的闭环。
- 规则版报告已经具备可追溯证据链。
- 代码级相似线索能帮助评审定位复核重点，但不会越权裁定。
- 样本来源等级明确，未核验比赛样本不会被包装成获奖案例。
- LLM 后续只作为受约束的报告润色器，不作为裁判。
